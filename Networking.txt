same network
Two or more devices are said to be in the "Same Network" when their network prefixes (determined by the Subnet Mask / CIDR block) are identical. They can communicate directly with each other at Layer 2 (Data Link Layer) using MAC addresses and ARP, without needing a router or a default gateway.
different network
different network
Devices are said to be in "Different Networks" when their network prefixes do not match. They cannot communicate directly. To send a packet from one network to another, it mandatory to pass through a Layer 3 device, which is a Router (or Default Gateway), to perform routing.

netstat -rn is a fundamental network troubleshooting command used to display the operating system kernel's Layer 3 routing table. It tells you exactly how your server routes data packets—whether a packet should stay inside the local subnet or be sent to an external router (Gateway) to reach the outside internet.
Breaking Down the Flags:
netstat: Network Statistics. A legacy CLI tool used for monitoring network connections and routing tables.
-r: Routing. Instructs the tool to fetch and display the kernel's internal IP routing table.
-n: Numeric. Forces the command to show raw IP addresses and ports instead of resolving them into human-readable hostnames (like localhost or gateway). This bypasses DNS lookups, making the command execute instantly.

LISTEN Port: The server is passively waiting on a specific port for new incoming connection requests (before the TCP 3-way handshake).

ESTABLISHED Port: The server has successfully completed the TCP 3-way handshake with a specific client, and an active, bi-directional data transfer channel is open.

Example Output:
Bash
Kernel IP routing table
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
0.0.0.0         192.168.1.1     0.0.0.0         UG    100    0        0 eth0
192.168.1.0     0.0.0.0         255.255.255.0   U     100    0        0 eth0
Understanding the Columns:
Destination: The destination network or host IP address.
Gateway: The next-hop router IP address.
0.0.0.0 means no gateway is required. The destination is in your Same Subnet (Local LAN), so the kernel talks directly to the machine via Layer 2 ARP.
An actual IP (e.g., 192.168.1.1) is your Default Gateway. External internet traffic is sent here.
Genmask: The subnet mask for the destination network.
0.0.0.0 with a destination of 0.0.0.0 denotes the Default Route (the entire outside internet).
Flags: Indicates the status of the route:
U (Up): The route is active.
G (Gateway): The route uses a gateway/router.
Iface (Interface): The specific network interface card (e.g., eth0) through which the packet will physically leave the server.
